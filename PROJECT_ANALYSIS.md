## 1. 项目概述

### 1.1 项目简介

**TinyZero** 是 [DeepSeek R1 Zero](https://github.com/deepseek-ai/DeepSeek-R1) 的复现项目，专注于**倒计时（Countdown）**和**乘法（Multiplication）**两个数学推理任务。该项目基于字节跳动的 [veRL](https://github.com/volcengine/verl) 框架构建。

### 1.2 核心价值

- **低成本验证**：只需 < $30 即可体验 DeepSeek-R1 Zero 的"Aha moment"
- **自主推理能力涌现**：通过强化学习，3B 基础语言模型能够自主发展出**自我验证**和**搜索**能力
- **无需监督数据**：使用纯强化学习训练，不需要人类标注的思维链数据

### 1.3 项目状态

> ⚠️ **注意**：该项目已不再积极维护，建议直接使用最新的 [veRL](https://github.com/volcengine/verl) 库。

## 3. 核心概念解析

### 3.1 Countdown 任务（倒计时游戏）

**任务描述**：给定一个目标数字和一组数字，使用基本算术运算（+、-、*、/）构造一个等于目标数字的等式，每个数字只能使用一次。

**示例**：
```
输入：
- 目标数字：100
- 可用数字：[25, 4, 3, 1]

输出：
- (25 - 1) * 4 + 3 = 99  ❌ 错误
- (25 + 3) * 4 - 12 = 100 ❌ 使用了不存在的数字
- 25 * 4 * (3 - 2 + 1) = 100 ❌ 使用了不存在的数字
- 25 * 4 = 100 ✅ 正确
```

### 3.2 PPO 算法（Proximal Policy Optimization）

PPO 是一种强化学习算法，用于训练语言模型。核心组件：

| 组件 | 作用 | 说明 |
|------|------|------|
| **Actor** | 策略网络 | 生成文本响应的语言模型 |
| **Critic** | 价值网络 | 评估生成文本的价值分数 |
| **Ref Policy** | 参考策略 | 防止模型偏离原始行为（KL约束） |
| **Reward Model** | 奖励函数 | 评估生成文本的质量 |

### 3.3 关键技术

1. **vLLM**：高性能推理引擎，用于快速生成响应
2. **FSDP**：完全分片数据并行，用于分布式训练
3. **Ray**：分布式计算框架，协调多GPU训练
4. **Flash Attention**：加速注意力计算

## 4. 训练流程详解

### 4.1 数据准备流程

```
原始数据集 (HuggingFace)
        ↓
  数据预处理脚本
  (countdown.py / multiply.py)
        ↓
    生成Prompt模板
        ↓
  保存为Parquet格式
        ↓
   train.parquet / test.parquet
```

**数据格式示例**：
```json
{
  "data_source": "countdown",
  "prompt": [{
    "role": "user",
    "content": "Using the numbers [25, 4, 3, 1], create an equation that equals 100..."
  }],
  "reward_model": {
    "style": "rule",
    "ground_truth": {
      "target": 100,
      "numbers": [25, 4, 3, 1]
    }
  }
}
```

### 4.2 训练主流程

```
┌─────────────────────────────────────────────────────────────────┐
│                        训练循环                                  │
├─────────────────────────────────────────────────────────────────┤
│  1. 加载数据                                                     │
│     └── 读取 train.parquet                                      │
│                                                                 │
│  2. Rollout 阶段（生成响应）                                      │
│     └── 使用 vLLM 快速生成多个响应                                │
│                                                                 │
│  3. 奖励计算                                                     │
│     └── 基于规则的奖励函数验证答案正确性                           │
│                                                                 │
│  4. 优势估计（GAE）                                              │
│     └── Critic 网络计算价值，结合奖励计算优势                      │
```

## 5. 核心代码模块分析

### 5.1 训练入口：`verl/trainer/main_ppo.py`

**核心职责**：
- 初始化 Ray 分布式环境
- 配置 Actor、Critic、RefPolicy 工作进程
- 定义奖励函数
- 启动 PPO 训练器

**关键代码**：
```python
@hydra.main(config_path='config', config_name='ppo_trainer', version_base=None)
def main(config):
    ray.init()  # 初始化Ray
    ray.get(main_task.remote(config))

@ray.remote
def main_task(config):
    # 定义角色-工作进程映射
    role_worker_mapping = {
        Role.ActorRollout: ray.remote(ActorRolloutRefWorker),
        Role.Critic: ray.remote(CriticWorker),
        Role.RefPolicy: ray.remote(ActorRolloutRefWorker)
    }

    # 创建奖励函数
    reward_fn = RewardManager(tokenizer=tokenizer, num_examine=0)

    # 创建训练器并开始训练
    trainer = RayPPOTrainer(config=config, ...)
    trainer.fit()
```

### 5.2 奖励管理器：`RewardManager`

**核心职责**：
- 根据数据源选择对应的奖励函数
- 解码模型输出并计算奖励分数

```python
class RewardManager():
    def __call__(self, data: DataProto):
        for i in range(len(data)):
            # 解码响应
            sequences_str = self.tokenizer.decode(sequences)

            # 选择奖励函数
            compute_score_fn = _select_rm_score_fn(data_source)

            # 计算分数
            score = compute_score_fn(solution_str=sequences_str, ground_truth=ground_truth)
            reward_tensor[i] = score
```

### 5.3 Countdown 奖励函数：`verl/utils/reward_score/countdown.py`

**评分逻辑**：

```
输入：模型生成的解决方案字符串

         ↓
    提取答案
    从 <answer>...</answer> 标签中提取方程

         ↓
    验证方程
    检查是否只使用了给定的数字，每个只用一次

         ↓
    计算结果
    使用安全的 eval() 计算方程结果

         ↓
    比较目标
    结果是否等于目标数字？

         ↓
    返回奖励
    - 正确：1.0
    - 格式正确但答案错误：0.1
    - 格式错误/无法解析：0
```

```
│  │  └─────────┘    └─────────────┘    └─────────────┘               │ │
│  └───────────────────────────────────────────────────────────────────┘ │
│                                    │                                    │
│                                    ↓                                    │

│                                    │                                    │

│  ┌───────────────────────────────────────────────────────────────────┐ │
│  │                     3. 价值估计阶段                                │ │
│  │  ┌─────────────┐    ┌─────────────┐    ┌─────────────┐           │ │

│  │  └─────────────┘    └─────────────┘    └─────────────┘           │ │
│  └───────────────────────────────────────────────────────────────────┘ │
```

## 7. 使用指南

### 7.1 环境安装

```bash
# 创建conda环境
conda create -n zero python=3.9

# 安装PyTorch
pip install torch==2.4.0 --index-url https://download.pytorch.org/whl/cu121

# 安装vLLM
pip3 install vllm==0.6.3
pip3 install ray

# 安装veRL
pip install -e .

# 安装Flash Attention
pip3 install flash-attn --no-build-isolation

# 可选：日志和可视化
pip install wandb IPython matplotlib
```

### 7.2 数据准备

```bash
conda activate zero
python ./examples/data_preprocess/countdown.py --local_dir ~/data/countdown
```

### 7.3 启动训练

**单GPU训练（≤1.5B模型）**：
```bash
export N_GPUS=1
export BASE_MODEL=~/models/Qwen2.5-1.5B
export DATA_DIR=~/data/countdown
export ROLLOUT_TP_SIZE=1
export EXPERIMENT_NAME=countdown-qwen2.5-1.5b
export VLLM_ATTENTION_BACKEND=XFORMERS

bash ./scripts/train_tiny_zero.sh
```

**多GPU训练（3B+模型）**：
```bash
export N_GPUS=2
export BASE_MODEL=~/models/Qwen2.5-3B
export DATA_DIR=~/data/countdown
export ROLLOUT_TP_SIZE=2
export EXPERIMENT_NAME=countdown-qwen2.5-3b
export VLLM_ATTENTION_BACKEND=XFORMERS

bash ./scripts/train_tiny_zero.sh
```

### 7.4 重要训练参数说明

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `data.train_batch_size` | 256 | 训练批次大小 |
| `data.max_prompt_length` | 256 | 最大提示长度 |
| `data.max_response_length` | 1024 | 最大响应长度 |
| `actor_rollout_ref.actor.optim.lr` | 1e-6 | Actor学习率 |
| `critic.optim.lr` | 1e-5 | Critic学习率 |
| `algorithm.kl_ctrl.kl_coef` | 0.001 | KL散度系数 |
| `trainer.total_epochs` | 15 | 总训练轮数 |
| `trainer.save_freq` | 100 | 保存频率 |

## 8. 技术细节

### 8.1 核心算法：PPO

**PPO目标函数**：
```
L^CLIP(θ) = E[min(r_t(θ) * A_t, clip(r_t(θ), 1-ε, 1+ε) * A_t)]
```

其中：
- `r_t(θ) = π_θ(a_t|s_t) / π_θ_old(a_t|s_t)` 是策略比率
- `A_t` 是优势函数
- `ε` 是clip参数（默认0.2）

### 8.2 GAE优势估计

```
A_t = Σ_l (γλ)^l * δ_{t+l}

其中 δ_t = r_t + γV(s_{t+1}) - V(s_t)
```

默认参数：`γ=1.0`, `λ=1.0`

### 8.3 KL散度约束

为防止模型偏离原始策略太远，使用KL散度作为正则化：

```
L_total = L_ppo + β * KL(π_θ || π_ref)
```

### 8.4 推理能力涌现现象

在训练过程中，模型会逐渐展现以下能力：

1. **自我验证**：检查自己的答案是否正确
2. **回溯搜索**：发现错误后尝试其他方法
3. **思维链**：逐步推理解决问题

这些能力**不是通过监督学习获得的**，而是通过强化学习**自主涌现**的。
